## 核心

`docs/文档体系建设/实践-v1/` 只记录文档体系建设和 docwarden 相关实践。

项目内 AI Infra 的实践不放在这里，放在：

```text
docs/ai基建建设/实践-v1/
```

这里关注的是：文档体系理论如何在 docwarden 项目中被实践出来。

## 当前实践关系

当前已经发生的 docwarden 相关实践是：

```text
.docwarden/task/
  -> review system
  -> spec / guide / wiki
```

这说明文档体系建设不只是写理论文档，也需要一个 agent 可以工作的项目内流程。

但这个流程本身属于 docwarden 的实践，不等于 AI Infra 的全部实践。

## 当前边界

本目录可以记录：

- docwarden 如何组织文档维护 workflow。
- `docs/` 作为 human-maintained source layer 的规则。
- `.docwarden/task` 如何承载文档体系建设过程材料。
- review system、spec、guide、wiki 如何服务文档体系。

本目录不记录：

- 项目内 AI Infra 的完整实践。
- custom skill creator 实现细节。
- CLI / plugin 的具体工程实现。
- contexta 的内容格式协议细节。

# docwarden

This repository is an early-stage workspace for two co-evolving product lines.

- `docwarden` is the primary project. It focuses on document systems, document-maintenance workflows, and the future CLI / plugin implementation for agent-era documentation.
- `contexta` is the AI infrastructure product line. It focuses on skills, schema, workflows, and context maintenance. It is designed from the start as infrastructure that can expand to the user level, not only as project-local tooling.

They are not split into separate projects yet because their ideas overlap heavily, their boundary is still unclear, and both are very early. For now, they develop through each other: `docwarden` protects and structures its own documents, while `contexta` supports the writing and iteration of `docwarden`.

TypeScript ESM CLI tool generated by create-yume with Effect.

## Scripts

```bash
pnpm install
pnpm build
pnpm smoke:bin
```

The build output is written to `dist/index.js`, with declarations at `dist/index.d.ts`.
The package exposes the `docwarden` npm bin at `dist/index.js`.
The CLI entrypoint uses `@effect/cli`, `NodeContext.layer`, and `NodeRuntime.runMain`.
